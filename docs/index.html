Installation

1

Install via Homebridge UI

Search for homebridge-esphome-waterlevel-hv2 in the Homebridge plugin tab and click Install.

2

Or install via npm

npm install -g homebridge-esphome-waterlevel-hv2
3

Restart Homebridge

The plugin will appear automatically after restart.

Configuration

Add the following to your Homebridge config.json, or use the Settings UI in the Homebridge dashboard.

{
    "platform": "esphome-hv2",
    "devices": [
        {
            "host": "192.168.1.50",
            "port": 6053,
            "encryptionKey": "YOUR_32_CHAR_ENCRYPTION_KEY"
        }
    ],
    "discover": true,
    "debug": false
}

FieldRequiredDescription
hostYesIP address or hostname of your ESP device (e.g. 192.168.1.50 or my_esp.local).
portNoNative API port. Defaults to 6053.
encryptionKeyNoNoise encryption key from your ESPHome YAML under api: encryption: key:.
passwordNoLegacy plaintext password. Only for older ESPHome setups.
discoverNoEnable mDNS auto-discovery of ESPHome devices on the network.
debugNoEnable verbose logging to the Homebridge console.

ESPHome YAML — Water Level Sensor Example

sensor:
  - platform: ultrasonic
    trigger_pin: GPIO5
    echo_pin: GPIO18
    name: "Water Tank Level"
    unit_of_measurement: "m"
    device_class: distance
    update_interval: 10s

api:
  encryption:
    key: "YOUR_32_CHAR_KEY_HERE"

Troubleshooting

Plugin does not appear after install

Make sure the platform value in your config is "esphome-hv2" and that you restarted Homebridge after installing.

"outdated API 0.0" error

This is a known issue with older plugins. HV2 fixes this by correctly negotiating the API version during the handshake.

Connection refused or no accessories appear

Verify that your encryptionKey matches exactly what is in your ESPHome YAML. Also ensure Homebridge can reach the device IP (try pinging it).

Water level sensor not showing up

Make sure your ESPHome sensor uses unit_of_measurement: "m", "cm", or "mm", or set device_class: distance. It will appear as a Light Sensor tile in the Home app — the numeric value is your distance reading.

Enable debug logging

Set "debug": true in your config. Detailed connection and entity discovery logs will appear in the Homebridge console.
